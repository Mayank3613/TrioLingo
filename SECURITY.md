# Security Policy

## Supported Versions

Currently, the following versions of TrioLingo++ receive security updates:

| Version | Supported |
| ------- | --------- |
| 0.1.x   | ✅ Yes     |
| < 0.1.0 | ❌ No      |

---

## Reporting a Vulnerability

We take the security of TrioLingo++ very seriously. If you find a security vulnerability, please do **not** open a public issue. Instead, report it using one of the following methods:

1. **GitHub Security Advisories**: Go to the **Security** tab of this repository on GitHub and select **Report a vulnerability** to privately submit a draft advisory.
2. **Contacting Maintainers**: Open a private discussion or contact the maintainers directly.

### What to Include in a Report
Please provide:
- A description of the vulnerability and its potential impact.
- Steps to reproduce the issue (and a proof of concept if possible).
- Any details on your system configuration (OS, browser, Tauri version).

---

## Disclosure Policy

Once a security vulnerability report is received:
- We will acknowledge receipt of your report within 48 hours.
- We will work on a fix as quickly as possible.
- A new release containing the security fix will be published, and credit will be given to the reporter (unless requested otherwise).
